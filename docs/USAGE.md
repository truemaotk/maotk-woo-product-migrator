# MaoTK Woo Product Migrator 使用教程

本教程适合宝塔服务器、WordPress 和 WooCommerce 站点。推荐先在临时站或测试站跑 50 个产品，确认正常后再批量迁移。

## 一、安装插件

1. 进入 WordPress 后台。
2. 打开 `插件` -> `安装插件` -> `上传插件`。
3. 上传 `maotk-woo-product-migrator.zip`。
4. 点击启用插件。
5. 左侧菜单会出现 `MaoTK 产品迁移`。

旧站和新站都需要安装这个插件。

## 二、旧站导出产品

1. 进入旧站后台。
2. 打开 `MaoTK 产品迁移`。
3. 先选择 `50` 个产品测试。
4. 如需自定义数量，在 `自定义数量` 输入产品数量。
5. 如果输入 `0`，表示全部导出。
6. 点击 `创建导出任务`。
7. 插件会一次只导出一个产品，完成一个再继续下一个。
8. 导出完成后，点击 `下载导出包`。

导出失败的产品不会阻塞后续产品。失败记录里会保存旧产品 ID、产品标题、SKU、旧产品链接和失败原因。

## 三、准备旧站图片目录

如果你有旧站完整备份，推荐在新服务器上解压旧站的：

```text
wp-content/uploads
```

例如可以放在：

```text
/www/wwwroot/old-site/wp-content/uploads
```

新站导入时填写这个路径，插件会优先从本地读取图片，不访问旧站网址。

这样可以避免：

- 旧站防盗链导致图片下载失败
- 旧站 WAF 或 CDN 拦截
- 旧站带宽太低
- 大量图片下载拖慢旧站

## 四、新站导入产品

1. 进入新站后台。
2. 打开 `MaoTK 产品迁移`。
3. 上传旧站导出的产品包。
4. 在 `旧站 uploads 目录` 填写旧站图片目录，例如：

```text
/www/wwwroot/old-site/wp-content/uploads
```

5. 选择重复产品处理方式。
6. 选择同名图片处理方式，推荐保持默认：

```text
同名图片自动加 mig-随机数字
```

7. 点击 `上传并创建导入任务`。
8. 插件会一次只导入一个产品，失败跳过，继续下一个。

## 五、重复产品怎么选

推荐优先按 SKU 匹配。

可选策略：

- `已有产品则跳过`：最安全，不改新站已有产品。
- `已有产品只更新价格和库存`：适合只同步价格库存。
- `已有产品更新全部字段`：会覆盖已有产品的标题、描述、分类、属性等。
- `已有产品也创建新产品`：不建议常用，容易产生重复产品。

## 六、同名图片处理

默认策略是自动重命名，不覆盖新站原图。

例如新站已经有：

```text
sofa.jpg
```

插件会保存成：

```text
sofa-mig-482193.jpg
```

然后创建新媒体库附件，并把产品主图、相册图、变体图绑定到新附件 ID。

## 七、失败处理

如果导出或导入有失败项，可以在历史任务里操作：

- `下载失败日志`
- `只重试失败项`
- `继续/运行`

失败日志包含：

```text
old_product_id,title,sku,old_url,stage,error_message,retry_count
```

重试失败项时，插件只会重新处理失败产品，不会重复处理成功产品。

## 八、建议顺序

1. 旧站导出 50 个产品。
2. 新站导入 50 个产品。
3. 检查产品标题、分类、属性、价格、库存、主图、相册图、变体。
4. 确认没问题后再导出 500 或 1000。
5. 最后再执行全部导出。

## 九、上线前检查

导入完成后建议检查：

- 产品数量是否正确
- 产品分类是否正确
- 主图和相册是否显示
- 可变产品的变体是否完整
- SKU 是否重复
- 前台产品页是否正常
- 新站缓存是否需要清理
- 固定链接是否需要重新保存

## 十、重要提醒

- 导入前请备份新站文件和数据库。
- 本插件迁移 WooCommerce 标准产品数据，不复制旧主题样式。
- Elementor、WPBakery、ACF、SEO 插件字段建议先小批量测试。
- 不建议直接把旧站数据库导入新站，否则会覆盖或污染新站内容。
